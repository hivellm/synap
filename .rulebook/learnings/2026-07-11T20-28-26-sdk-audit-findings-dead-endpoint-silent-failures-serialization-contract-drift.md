# SDK audit findings: dead-endpoint silent failures + serialization contract drift
**Source**: manual
**Date**: 2026-07-11
phase14 SDK 1.0 audit across 7 SDKs (rust/ts/python/php/csharp/go/java). Findings worth remembering: (1) The Python SDK's KV module (the most-used module!) still called a legacy /api/stream endpoint the server removed — and because the client treated an empty HTTP response as {}, EVERY kv.* call silently returned None instead of erroring. Silent-empty fallbacks hide dead integrations; probe SDK modules against a LIVE server, not just unit mocks. (2) TS kv.set leaked objects as "[object Object]" (String coercion in the command mapper) while kv.get documented JSON auto-parse — write-side and read-side contracts drifted. Fixed by JSON.stringify for non-string values on set in BOTH TS (stored()) and Python (_encode_value), making the JSON round-trip contract symmetric across SDKs. (3) Mock-based transport tests rot: Python's mocks asserted an array-framed msgpack request while the SDK actually sends map-framed (rmp-serde on the server accepts both) — the mocks failed while the real integration worked; when a mock and a live test disagree, trust the live one and fix the mock. (4) Auth rejection tests need a runtime probe (does this server enforce auth?) + skip, or they fail against dev servers. (5) Dependency knowledge: npm lockfile ERESOLVE when a meta-package (typescript-eslint) pins exact versions of its plugins — update all three together; uuid v7+ ships own types (drop @types/uuid); TypeScript 7 is the native-compiler major (hold on stable branches); .NET analyzers version should match the SDK major. All 7 SDKs default to SynapRPC (synap://host:15501) with RESP3/HTTP via URL scheme.
